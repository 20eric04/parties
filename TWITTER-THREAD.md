## Production Notes

**Video placement:**
- Version A: Tweet 4 (after the problem setup, so the product demo lands as the payoff)
- Version B: Tweet 2 (lead with the product immediately after the hook — let people see it right away)

**Posting strategy:**
- Post tweet 1, then reply-chain the rest with ~60 seconds between each
- Pin the thread to your profile for launch week
- Quote-tweet the CTA (last tweet) from the main @parties account if you have one

**What to film for the video (30-45 seconds):**
- Screen recording of the app: search Miami, tap a venue, build a multi-booking (table + dinner + car), checkout screen
- Dark UI will look great in-feed — the #E54D2E accent pops on dark mode Twitter
- No voiceover needed. Captions or text overlay work better for autoplay

**Optional adds:**
- If you have a photo of a real venue (LIV, E11EVEN), reply to the venue tweet with it
- A "building in public" follow-up tweet a few days later with a Supabase dashboard screenshot or Stripe test payment will get dev engagement
